# 冒烟测试 — 主路径

从登录到订单生成的完整 happy path。

## 自动化

```bash
uv run pytest backend/tests/test_smoke_flows.py -m smoke_main -q
PLAYWRIGHT_SKIP_WEBSERVER=1 pnpm exec playwright test --config playwright.config.ts e2e/main-path.spec.ts
```

## 手动步骤

### 0. 鉴权与会话

| # | 操作 | 预期 |
|---|------|------|
| 1 | 打开 `http://localhost:5173` | 登录浮层 |
| 2 | 登录或注册 | 进入主页 |
| 3 | 点击「规划新行程」 | 新建会话，助手问候 |

### 1. 需求收集 `collect_requirements`

| # | 输入 | 预期 |
|---|------|------|
| 1 | `北京` | 追问出发城市 |
| 2 | `上海` | 追问出发日期 |
| 3 | `2026-06-19` | 识别端午节；追问天数 |
| 4 | `3天` | 追问人数；**无**预算「一般党」 |
| 5 | `就我一个人` | 追问预算 |
| 6 | `穷游党` | 单段需求确认（不重复摘要） |
| 7 | `对的` | 进度 → **目的地** |

### 2. 目的地 `plan_destination`

| 情况 | 操作 | 预期 |
|------|------|------|
| 已确认目的地 | 可能自动推进 | 「目的地已确认：北京」→ **交通** |
| 需确认 | `就北京` | 进入交通阶段 |

### 3. 交通 `plan_transport`

| # | 输入 | 预期 |
|---|------|------|
| 1 | 阅读对比 | 可出现工具查询提示 |
| 2 | `高铁` | 「交通方式已确认：高铁」→ **食宿** |

### 4. 食宿 `plan_stay_and_food`

| # | 输入 | 预期 |
|---|------|------|
| 1 | `经济酒店，本地小吃` | → **活动** |

### 5. 活动 `plan_activities`

| # | 输入 | 预期 |
|---|------|------|
| 1 | `文化体验` | 「开始生成完整行程」→ **行程** |

### 6. 行程 `build_itinerary`

| 检查项 | 预期 |
|--------|------|
| 助手回复 | 逐日行程 + 预算拆分 |
| 右侧卡片 | `ItineraryCard` 可见 |
| 底部横幅 | 「行程已生成，请确认或提出修改」 |

### 7. 确认 `approval_node`

| # | 操作 | 预期 |
|---|------|------|
| 1 | 点击「确认行程」或发送 `确认` | → **完成** |

### 8. 订单 `final_response`

| 检查项 | 预期 |
|--------|------|
| 回复 | 含 `ORDER-XXXXXXXX`、目的地、预算 |

## Checklist

```
□ 8 个进度阶段均可到达
□ 需求顺序：目的地→出发城市→日期→天数→人数→预算→确认
□ 无提前出现的预算/旅行风格
□ 行程卡片渲染正常
□ 生成订单号
```
